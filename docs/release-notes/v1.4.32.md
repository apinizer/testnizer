## v1.4.32

**Imported test suites now run end-to-end, your open tabs survive an app
restart, and the REST workbench gets a polish pass.** Closes fifteen reported
issues.

### Test Suites

- **Imported suites run correctly:** importing an OpenAPI / Postman / Insomnia
  collection as a test suite now preserves nested folders and the auth + pre/post
  script cascade, so a request inheriting a folder bearer token or a setup script
  runs the same way it did in the source tool instead of failing with a 401 or a
  "proxy not found". Variables and tokens derived in a Setup step flow through to
  later requests.
- **Quick Run:** run a suite folder straight from the tree without stepping
  through the config screen first.
- **Folder settings:** edit a suite folder's auth and cascade pre/post scripts
  from a dedicated settings dialog.
- **Save-before-run guard:** a request edited but not saved is flushed before the
  runner executes it, so it never runs against its stale saved snapshot — on both
  the configured-run and Quick Run paths.
- **Export a suite** to Postman v2.1 or Insomnia v4 JSON, not just the native
  format.

### Request workbench

- **Open tabs survive a restart (#31):** the tabs you had open — and the request
  values inside them — are restored when you reopen the app. They were being
  wiped on the first project open because the per-project tab cache started empty
  on boot; it now persists across launches.
- **No stray scrollbar on the response tab strip (#34):** the Body / Cookies /
  Headers row no longer shows a phantom vertical scrollbar.
- **Theme-correct request tabs (#32, #33):** the request tab uses the accent
  colour and underline like the response pane, and the two panes' selected tabs
  line up.
- **Sentence-case response Headers (#35):** the response Headers columns read
  Key / Value, matching the request Headers table.
- **URL bar scales with the font size (#38):** the method selector and address no
  longer sit at a fixed size that overflowed at larger font scales.
- **Resizable Params / Headers columns (#36, #37):** the request Query Params and
  Headers table columns can be dragged to resize.
- **Collapse / expand all folders (#39):** one click toggles every folder in the
  APIs tree.
- **Rename updates the tab title (#40):** renaming an API in the tree updates its
  open tab immediately.
- **Ctrl+S saves in place (#41):** the shortcut saves an open endpoint tab like
  the Save button, instead of popping the Save As modal.
- **Muted table placeholders (#42):** empty-row placeholders are no longer
  emphasised like column headers.

### Requests & runner

- **Pre-request env writes persist across sends (#29):** a value written from a
  pre-request script (e.g. a derived `ApiProxyName`) is kept for the requests
  that follow, so policy requests stop failing with "proxy not found".
- **Idempotent DELETE counts as passed (#30):** a non-2xx response with a passing
  `pm.test` (e.g. a DELETE asserting `oneOf([200, 400])`) is reported as Passed,
  not Failed.
- **Import from clipboard (#28):** paste an OpenAPI / Postman / Insomnia spec
  straight from the clipboard in the import dialog.

### Tools

- **Text Diff find & navigation:** the diff result gains an in-result find bar
  (Ctrl/Cmd+F with match highlighting) and previous / next difference buttons.

**Tests:** import→run end-to-end guards for Postman and Insomnia (nested folders,
folder-cascade auth/scripts, cross-request derivation, cookie-jar projectId),
round-trip suite export → re-import for Postman / Insomnia, OpenAPI plural
examples, Quick Run save-before-run, and a relaunch tab-restore regression
(`project-tabs-persist`).

**Still open:** #27 (the 1.4.30 x64 installer crash during setup) is a separate
Windows packaging investigation and is not covered by this build.
